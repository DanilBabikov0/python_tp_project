Техническое задание на проект: Генератор Палитр

1. Цель проекта
    Создание веб-приложения для генерации цветовых палитр на основе заданного цвета или изображения. Система предоставляет пользователям возможность:
        Генерировать палитры по цветовому кругу (монохромные, аналоговые, комплементарные, триадные)
        Извлекать цвета из загруженных изображений
        Сохранять, просматривать и управлять своими палитрами
        Авторизовываться через Google

2. Роли пользователей
    Гость: может генерировать палитры по цвету, просматривать результаты, копировать цвета, использовать все базовые функции
    Авторизованный пользователь: имеет все функции гостя, а также может сохранять палитры, просматривать свои сохраненные палитры, удалять их, извлекать цвета из изображений
    Администратор: полный доступ к административной панели Django, управление пользователями и данными

3. Модели данных (сущности)
    Palette (Палитра): Сохраненные пользовательские палитры, созданные по цвету
        Поля: id, name, colors (JSONField), owner (ForeignKey to User), created_at, updated_at
    ImagePalette (Палитра из изображения): Палитры, извлеченные из загруженных изображений
        Поля: id, image_name, preview (FileField), colors (JSONField), owner (ForeignKey to User), created_at, updated_at
    ExtractUsage (Лимит извлечения): Отслеживание количества извлечений цветов из изображений пользователем
        Поля: id, user (ForeignKey to User), count, last_used

4. Ключевой функционал (User Stories)
    Генерация палитры по цвету: Пользователь вводит цвет в формате HEX или выбирает через цветовую палитру, система генерирует 4 типа палитр с визуальным отображением
    Извлечение цветов из изображения: Пользователь загружает изображение, система извлекает заданное количество доминирующих цветов с предварительным просмотром
    Сохранение палитр: Авторизованный пользователь может сохранить любую сгенерированную палитру под своим именем, с возможностью просматривания и удаления
    Управление палитрами: Пользователь может просматривать, удалять и копировать свои сохраненные палитры
    Аутентификация через Google: Пользователь может войти через Google
    Копирование цветов: Пользователь может копировать отдельные цвета или все цвета из палитры в буфер обмена
    Случайная генерация: Возможность генерировать случайные палитры всех типов или отдельных типов
    Управление именами палитр: Пользователь может задавать собственные имена для сохраняемых палитр

5. Внешние интеграции и аналитика
    Интеграция с Google OAuth: Использование django-allauth для аутентификации через Google с автоматическим заполнением данных
    Анализ цветов из изображений: Использование библиотек colorthief и Pillow для извлечения цветов из изображений
    Генерация палитр: Использование собственных алгоритмов на основе цветового круга и HSL-модели для генерации различных типов палитр
    Лимиты запросов: Реализация ограничений на количество извлечений цветов из изображений в определённый промежуток времени (10 извлечений за 60 минут)
    Уведомления и интерфейс: Реализация пользовательских уведомлений, drag-and-drop загрузки, визуального отображения цветов
    CSS-стилизация: Использование швейцарского дизайна с использованием шрифта Inter, иконок и современных стилей

6. Техническая архитектура
    Backend: Django с использованием Django REST Framework для API
    Frontend: HTML/CSS/JavaScript
    База данных: SQLite (возможность миграции на PostgreSQL)
    Аутентификация: Django-allauth с Google OAuth
    Файлы: Хранение превью изображений в директории /previews/
    Статика: Отдельные директории для CSS, JS, изображений и иконок
